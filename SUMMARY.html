Flaky Network Assignment: Architecture Summary

1. Core Strategy

The goal was to reliably transport 160-byte audio frames across a lossy, jittery UDP relay while enforcing a strict 60ms deadline and a 2.0x bandwidth cap. To achieve this, the architecture uses a hybrid FEC (Forward Error Correction) + ARQ (Automatic Repeat Request) design governed by a strict token bucket.

2. Bandwidth Compression

Because every byte counts towards the 2.0x cap, protocol overhead was aggressively minimized:

  • 16-bit Sequence Numbers: The harness's 32-bit sequence numbers are compressed to 16 bits internally, saving 2 bytes per packet. The receiver automatically resolves wraparound epochs.
  • Relative FEC Offsets: Instead of transmitting an absolute sequence number for the backup frame, the protocol sends a 1-byte relative offset (seq - offset), saving an additional 1-3 bytes.

3. Forward Error Correction (FEC)

Every "full" packet (323 bytes) carries both the current audio frame and a redundant backup of a previous frame. We set the FEC offset to K=1. When frame N is dropped, its backup rides along in frame N+1. Frame N+1 is sent 20ms later, leaving 40ms of headroom to absorb network jitter before crossing the 60ms deadline. This recovers ~90% of lost packets flawlessly.

4. Automatic Repeat reQuest (ARQ)

To catch burst drops where FEC fails (e.g., both frame N and N+1 are dropped), the receiver actively monitors for gaps. If a gap exceeds 40ms or 2 frames, it fires a 2-byte NACK back to the sender.

The sender catches these NACKs and pushes them into a FIFO queue. Every 20ms tick, instead of sending its standard FEC backup, it serves one NACK from the queue, substituting the requested frame into the FEC payload slot. Retries are strictly capped at 2 to prevent infinite NACK loops.

5. The Token Bucket Limiter

A token bucket governs the sender to guarantee it never breaches the 2.0x bandwidth cap. It earns 310 bytes per frame tick and caps at 1000 bytes. When a NACK is received, 2 bytes are instantly deducted. If the budget falls below 323 bytes, the sender downgrades to a "small packet" (162 bytes, no FEC) to regenerate its balance.
